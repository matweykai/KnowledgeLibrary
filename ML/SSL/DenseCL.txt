DenseCL - это расширение MoCO для задач детекции и сегментации

* Также как и в MoCo тут присутствует схема с 2 энкодерами и очередью

* Именно в этой статье был введён термин dense contrastive learning, который
  обозначает класс задач, в которых SSL применяется для предобучения бэкбонов
  для задач детекции и сегментации

* В статье рассматриваются свёрточные энкодеры, в которых используются 2 головы:
  одна из них работает с глобальным представлением картинки (вектор после
  AvgPooling слоя), а другая работает с картой признаков размера S_w * S_h

* В данном подходе используется комбинированный функционал потерь:

	loss = lambda * dense_loss + (1 - lambda) * global_loss

* Для сравнения карт признаков применяется функционал потерь аналогичный InfoNCE
  из MoCo:

	dense_loss = 1 / s^2 * sum(-log( 
		exp(r_s * t_+_s / t) / 
		(exp(r_s * t_+_s / t) + sum(exp(r_s * t_-_s / t)))
	) )
	
	r_s - вектор в одной из ячеек карты признаков
	t_+_s - похожий на r_s вектор
	t_-_s - вектор непохожий на r_s (из другой картинки)
	t - температура

* Для нахождения t_+ для s используется косинусная близость между картами
  данного изображения и другой его аугментации
